# Claude Code 워크숍 진행 가이드 (3시간)

> 비전공자 강사가 비전공자 2명에게 가르치는 친밀한 워크숍

---

## 0. 한눈에 보기

**학습자**
- **학습자 A — 작가/팟캐스터 (이하 "작가님")**: 베스트셀러 작가, 자기 브랜드 팟캐스트 운영. 콘텐츠 제작·발행·청취자 소통이 일상. 디지털 친숙도 중간(글쓰기 도구·SNS는 능숙, 터미널은 처음일 가능성).
- **학습자 B — 1인 PT샵 운영자 (이하 "트레이너님")**: 회원 관리·운동/식단 프로그램 작성·예약 관리·인스타 콘텐츠 제작이 일상. 디지털 친숙도 중간(인스타·카톡은 능숙, 코드는 두려움 가능성).

**학습 목표 (워크숍 끝나고 학습자가 할 수 있어야 하는 것)**
1. Claude Code가 무엇이고 ChatGPT와 어떻게 다른지 한 문장으로 설명할 수 있다.
2. 자기 노트북에서 Claude Code를 실행하고 자기 일에 관련된 작업 한 가지를 시켜본 경험이 있다.
3. "하네스"라는 개념을 자기 언어로 설명할 수 있고, 자기 일에 적용할 첫 아이디어 한 가지를 가지고 돌아간다.

**총 시간:** 3시간 (180분)
**참가자 수:** 2명 (+강사 1명)
**준비물:** 노트북(맥/윈도우), 사용 중인 Claude 계정 또는 Anthropic API 키, Wi-Fi, 멀티탭, 음료/간식

### 워크숍 1주 전

### 워크숍 1일 전
- [ ] 강사 본인 노트북에서 Claude Code 깨끗이 실행 확인

- [ ] 학습자별 첫 프로젝트 시나리오(§3) 강사가 한 번 직접 돌려보기

### 워크숍 1시간 전
- [ ] 멀티탭·음료 세팅
- [ ] 빔/모니터 미러링 테스트
- [ ] 강사 노트북 알림 OFF, 데스크탑 정리
- [ ] HANDOUT.md 인쇄 또는 PDF 공유 준비

## 2. 분 단위 진행 가이드

### 0–15분 │ 오프닝 + 아이스브레이커
**목적:** 긴장 해소, "나도 비전공자다" 선언으로 심리적 안전감 형성, 기대 정렬

**진행:**
1. (3분) 환영 인사. 강사 자기소개 — **반드시 "저도 비전공자입니다"라고 말한다.** "그런데 6개월 전에 이걸 배워서 제 일이 이렇게 바뀌었어요"라고 짧은 변화 1가지 공유.
2. (6분) 두 학습자 자기소개 + **아이스브레이커 질문 1개**:
   > **"요즘 일하면서 '아 이거 누가 대신 해줬으면 좋겠다' 싶은 일 한 가지만 말씀해주세요."**
   - 강사는 화이트보드/노션에 받아 적는다. **이게 90분 후 실습 프로젝트의 씨앗이 된다.**
3. (4분) 오늘의 약속 3가지를 공유:
   - "정답 없습니다. 막히면 손드세요. 막힌 게 가장 좋은 학습 순간입니다."
   - "오늘 코드 한 줄도 직접 안 쓸 수도 있어요. 그게 이 도구의 핵심입니다."
   - "오늘 끝나고 여러분이 자기 일에 쓸 수 있는 작은 거 하나 만들어 가는 게 목표입니다."
2. (2분) 오늘 타임라인 한 장으로 보여주기

**강사 멘트 예시:**
> "여러분, 이거 코딩 배우는 워크숍이 아니에요. 코딩을 안 하고도 컴퓨터한테 일을 시키는 법을 배우는 워크숍입니다. 저도 여전히 코드 못 짭니다. 근데 이거 배우고 나서 제 팟캐스트 대본 정리하는 데 쓰던 2시간이 10분이 됐어요."

**주의:** "프로그래밍" "터미널" 같은 단어를 첫 15분 안에는 쓰지 말 것.

---

### 15–40분 │ Claude Code란? + 강사 라이브 시연
**목적:** "와 이게 되네" 충격 제공. 학습자가 "나도 해볼래" 모드로 전환되게 하기.

**진행:**
1. (5분) **Claude Code 한 문장 설명:**
   > "ChatGPT가 채팅창에 답해주는 친구라면, Claude Code는 **내 컴퓨터에 들어와서 직접 일을 해주는 친구**예요. 파일을 만들고, 폴더를 정리하고, 글을 고치고, 자료를 모아옵니다."
   - ChatGPT vs Claude Code 비교를 화이트보드에 그림으로:
     ```
     ChatGPT  →  말로만 답함  →  복사 붙여넣기는 내가
     Claude Code → 내 컴퓨터 안으로 들어옴 → 파일·폴더를 직접 만짐
     ```

2. (15분) **강사 라이브 시연 — wow 모먼트 3개:**

   **Wow #1 (5분):** 강사가 Claude Code를 켜고 자연어로 명령:
   > `"내 데스크탑에 'workshop' 폴더 만들고 그 안에 hello.txt 파일 만들어서 '안녕하세요'라고 써줘"`
   - 학습자에게 결과를 직접 보여줌. 폴더가 진짜 생기는 것을 본다.

   **Wow #2 (5분):** 더 의미있는 작업 시연:
   > `"내 다운로드 폴더에 있는 PDF 파일들의 제목만 모아서 마크다운 표로 정리해줘"`
   - **여기서 학습자들이 "엇" 하고 반응함.**

   **Wow #3 (5분) — 일부러 실패하기:** 강사가 모호한 지시를 내려본다:
   > `"이거 좀 정리해줘"`
   - Claude가 "어떤 걸 어떻게 정리할까요?"라고 되묻는 모습을 보여준다.
   - **교훈을 칠판에:** "Claude는 마법이 아니라 똑똑한 동료입니다. 동료한테 지시하듯이 구체적으로 말해야 해요."

3. (5분) Q&A. 보통 첫 질문은 "이거 돈 들어요?" — 솔직하게 답한다.

**강사 멘트 예시 (시연 중):**
> "지금 제가 한 거 보셨죠? 코드 한 줄도 안 썼어요. 한국어로 부탁만 했어요. 이게 핵심입니다."

**주의:** 시연할 때 **너무 화려한 거 보여주지 말 것.** 학습자가 "나는 절대 못 해" 모드로 들어가면 망친다. 일상적이고 작은 것 위주.

---

### 40–75분 │ 설치 + Hello World (학습자 직접)
**목적:** **"내 손으로 첫 성공"** — 이 35분이 워크숍의 운명을 결정한다.

**진행:**
1. (5분) 강사가 화면에 설치 명령 한 줄을 띄운다 (**미리 메모장에 복붙용으로 준비**):
   - 맥: `npm install -g @anthropic-ai/claude-code` (또는 가장 최신 공식 설치 명령)
   - 학습자에게 "이 한 줄만 따라하면 됩니다"라고 말한다.

2. (15분) **설치 단계 — 학습자 한 명씩 강사가 옆에 붙어서 진행.**
   - 다른 학습자는 그동안 자기 사용 사례 메모를 적어둔다 (지루하지 않게)
   - **흔한 막힘 포인트:**
     - Node.js 미설치 → 강사가 미리 준비한 한 줄짜리 안내로 즉시 처리
     - 권한 오류 → `sudo` 한 줄 안내 (의미는 30초만 설명: "관리자 권한으로 설치한다는 뜻")
     - 윈도우의 경우 PowerShell vs CMD 혼동 → PowerShell 사용 통일

3. (5분) **첫 실행:** 두 학습자가 동시에 `claude` 입력 → 환영 화면 본다.
   - **여기서 박수!** 강사가 솔직하게 "이게 제일 큰 산이었어요. 지금부터는 쉽습니다" 선언.

4. (10분) **Hello World — 자기 일과 살짝 연결:**
   - 작가님: `"오늘 워크숍 참가 후기를 3문단으로 써줘. 톤은 친근하게."`
   - 트레이너님: `"오늘 시작한 새로운 도구 워크숍 후기를 인스타 캡션 스타일로 써줘. 해시태그도 포함해서."`
   - **각자 결과를 옆 사람한테 보여준다.** (첫 공유 모먼트)

**강사 멘트 예시 (막힐 때):**
> "지금 막히신 거 진짜 좋은 거예요. 이 막힘을 한 번 겪어두면 혼자 할 때도 안 무서워져요. 자, 같이 해결해봅시다."

**주의:** 한 명이 막히면 다른 한 명이 기다리느라 지친다. **두 사람이 거의 동시에 진행되도록 페이스 조절.** 많이 차이나면 빠른 사람한테 "후기 한 번 더 다른 톤으로 써보세요"라고 시킨다.

---

### 75–85분 │ 휴식
**목적:** 인지 리셋. 강사도 쉬지만 1분만 쓴다.

**진행:**
- 음료, 화장실
- **강사는 이 휴식 시간에 두 학습자의 ‘아 이거 누가 대신…' 메모를 보고 90분 실습 프로젝트를 한 번 더 머릿속으로 정리한다.**
- 학습자끼리 자연스럽게 얘기하도록 두기 (개입 X)

---

### 85–110분 │ 하네스 엔지니어링 + 두 번째 시연
**목적:** "이걸 내 일에 진짜 쓸 수 있겠다"는 두 번째 wow. 다음 90분 실습의 도화선.

**진행:**
1. (5분) **하네스가 뭔지 비유로 설명:**
   > "Claude는 똑똑한 인턴이라고 생각해보세요. 매번 처음부터 가르치면 지치겠죠? 하네스는 이 인턴한테 '너는 우리 회사에서 이런 일을 이렇게 하는 사람이야'라고 미리 정리해주는 매뉴얼 같은 거예요."
   - 칠판에 그림:
     ```
     [Claude Code] ← 매번 새로 설명     vs     [Claude Code + 하네스]
                                                       ↑
                                                  내 일을 미리 알고 있는 친구
     ```

2. (5분) **하네스 = 3가지 재료** (이름은 안 외워도 됨, 개념만):
   - **CLAUDE.md** = 인턴한테 주는 "우리 회사 안내서" (내가 누구고, 뭘 좋아하고, 어떻게 일하는지)
   - **Skills** = "이런 일은 이렇게 해라" 매뉴얼
   - **Agents** = "이 일은 저 동료한테 시켜라" 역할 분담

3. (15분) **두 번째 라이브 시연 — 학습자별 미니 하네스:**
   - 강사 노트북에 미리 만들어둔 작가님용 CLAUDE.md를 보여준다 (5–10줄):
     ```markdown
     # 나의 글쓰기 스타일
     - 베스트셀러 에세이 작가
     - 톤: 친근하고 따뜻함, 거리감 없는 반말 X
     - 자주 쓰는 단어: ...
     - 절대 쓰지 않는 단어: ...
     - 글 마다 마지막에 짧은 질문으로 끝낸다
     ```
   - 그 다음 Claude한테 평범한 부탁을 한다: `"이 주제로 짧은 글 써줘: 봄이 오면 시작하기 좋은 한 가지"`
   - **CLAUDE.md가 있을 때 vs 없을 때**의 결과를 나란히 보여준다.
   - **이게 wow #2다.** "어 이거 진짜 작가님 스타일인데?"

**강사 멘트 예시:**
> "지금부터 90분 동안 여러분이 하실 일이 정확히 이거예요. 자기만의 작은 하네스 하나를 만들어서, Claude가 여러분을 알고 일하게 만드는 거. 어렵지 않아요."

---

### 110–160분 │ 자기 프로젝트 실습 (50분)
**목적:** 워크숍의 핵심. 각 학습자가 자기 일과 직결된 작은 산출물 1개를 손에 들고 나가게 한다.

> 상세 시나리오는 §3 참조. 강사는 두 학습자 사이를 오가면서 짧게 코칭. 한 사람한테 5분 이상 붙어있지 말 것.

**대원칙:**
- 학습자가 직접 프롬프트를 입력한다. 강사가 키보드 잡지 말 것.
- 막히면 **답을 주지 말고 질문으로 유도:** "Claude한테 뭐라고 물어보면 좋을까요?"
- 30분 지점에 한 번 모여서 **중간 공유 (3분):** 각자 지금까지 뭐했는지 말로 한 줄 요약. 자극받게 하기.

---

### 160–180분 │ 공유 + 마무리 + 다음 한 걸음
**목적:** 결과물 자랑 → 자기효능감. 다음 약속 → 안착.

**진행:**
1. (10분) **결과물 발표 (각 5분):**
   - 자기가 만든 것 화면 공유
   - "이걸 만들면서 제일 신기했던 순간"
   - **사진/스크린샷 찍기.** 기념 모먼트.
   - 강사 + 다른 학습자가 박수와 구체적 피드백 ("그 톤 진짜 작가님답네요" / "이거 회원분들이 좋아하시겠어요")

2. (5분) **HANDOUT.md 함께 살펴보기:**
   - 오늘 배운 명령 5개
   - 막혔을 때 시도할 3가지 ("재시작 / 더 구체적으로 / 강사한테 카톡")
   - 다음 한 걸음 (각 학습자별 맞춤 1개)

3. (5분) **마무리 의식 — 가져갈 한 문장:**
   - 각자 한 문장으로 "오늘 워크숍에서 가져가는 한 가지"
   - 강사가 받아 적어서 1주일 후 카톡으로 다시 보내준다 (**약속하기**)
   - 후속 채널 안내 (오픈 카톡방이든 그냥 강사 개인 카톡이든)
   - "1주일 안에 막히면 무조건 연락하세요. 그 막힘을 같이 푸는 게 진짜 학습입니다."

**강사 멘트 예시 (마무리):**
> "오늘 여러분이 만든 거, 작아 보여도 진짜 큰 거예요. 6개월 전 저는 이거 못 했어요. 오늘 여러분은 했어요. 다음 주에 카톡으로 안부 물을게요."

## 3. 학습자별 실습 시나리오 (110–160분, 50분)

### 학습자 A — 작가님: "내 톤으로 글 써주는 글쓰기 파트너 만들기"

**왜 이 사람에게:** 작가의 가장 큰 페인은 "내 톤을 모르는 AI가 평범한 글을 뱉는 것". 이걸 한 번에 해결하는 경험은 압도적 wow.

**최종 산출물:** `~/writing/` 폴더 + 그 안의 `CLAUDE.md` (작가님의 스타일 가이드) + 예시 글 1편

**분 단위:**
- **0–10분 │ 폴더 + CLAUDE.md 만들기**
  - 프롬프트 예: `"내 홈에 'writing'이라는 폴더를 만들고, 그 안에 'CLAUDE.md'라는 파일을 만들어줘. 내가 글쓰기 스타일을 적을 거야."`
  - 강사는 옆에서 CLAUDE.md를 같이 채워준다 — 작가님이 말하면 강사가 받아 적게 유도. 7–10줄.
- **10–25분 │ 첫 글 써보기**
  - 프롬프트 예: `"이 주제로 1500자 에세이를 써줘: [작가님이 직접 고른 주제]"`
  - 결과를 함께 읽고 어색한 부분 발견. **"이게 왜 어색하지?"** → CLAUDE.md를 한 줄 추가 → 다시 써달라고 요청.
  - **이 반복이 wow 모먼트.** "아, 이렇게 학습시키는 거구나"
- **25–40분 │ 팟캐스트 쇼노트 자동화 미니 실험**
  - 프롬프트 예: `"내가 30분 분량 팟캐스트 한 회 녹음을 했다고 가정하고, 가상의 에피소드 제목과 5문장짜리 쇼노트를 내 톤으로 써줘. 마지막에 청취자 질문 1개로 끝내줘."`
  - **여기가 진짜 wow.** 작가님이 "이거 진짜 내 팟캐스트 같아"라고 반응하는 순간.
- **40–50분 │ 정리 + 다음 한 걸음**
  - CLAUDE.md를 다듬는다.
  - 강사가 보여줌: "다음 주에 진짜 팟캐스트 녹음하시면, 텍스트 받아쓰기 파일을 이 폴더에 넣고 똑같은 톤으로 쇼노트 만들어달라고 하시면 돼요."

**막히면 강사 힌트:**
- "톤이 어색해요" → "어떤 단어가 어색했어요? 그 단어를 CLAUDE.md에 '안 쓰는 단어'로 추가해봅시다"
- "글이 너무 길어요" → "Claude한테 '이 글을 절반으로 줄여줘'라고 다시 말해봅시다"

**백업 프로젝트:** 작가님의 책 1권 목차를 주고, 각 챕터별 핵심 메시지를 한 문장으로 요약한 마크다운 만들기

### 학습자 B — 트레이너님: "회원별 운동 프로그램 만들어주는 PT 어시스턴트"

**왜 이 사람에게:** 1인 PT샵의 최대 페인은 "회원마다 다른 프로그램을 매주 새로 짜는 시간". 이걸 자동화하는 첫 발은 강력하다.

**최종 산출물:** `~/pt-assistant/` 폴더 + `CLAUDE.md` (트레이너님의 코칭 스타일·기본 원칙) + `members/` 폴더 + 회원 1명의 1주일 운동 프로그램 마크다운

**분 단위:**
- **0–10분 │ 폴더 구조 + CLAUDE.md 만들기**
  - 프롬프트 예: `"내 홈에 'pt-assistant' 폴더 만들고, 그 안에 CLAUDE.md, members 폴더 만들어줘"`
  - CLAUDE.md에 적을 것 (강사 유도):
    - 트레이너님 자격증/경력
    - 코칭 철학 (예: "안전 먼저, 자세 먼저")
    - 절대 안 시키는 운동 (예: 어떤 부상 위험 동작)
    - 한 회당 보통 시간, 운동 종류 분배
- **10–20분 │ 가상의 회원 1명 만들기**
  - 프롬프트 예: `"members 폴더 안에 'kim-yujin.md' 파일 만들고, 가상의 30대 여성 회원 정보를 넣어줘 — 운동 목표는 체지방 감량, 일주일에 2회 PT, 무릎이 살짝 약함, 운동 경력 6개월"`
- **20–40분 │ 1주일 프로그램 자동 생성**
  - 프롬프트 예: `"kim-yujin 회원의 이번 주 PT 2회 운동 프로그램을 짜줘. 회원 정보랑 내 코칭 원칙을 참고해서. 표 형식으로 운동 이름, 세트, 휴식시간 다 들어가게."`
  - **결과를 같이 본다.** 트레이너님이 "어, 이거 무릎 약한 사람한테는 안 좋은데"라고 발견 → CLAUDE.md에 한 줄 추가 → 다시 짜달라고 한다.
  - **이 학습 루프가 wow 모먼트.** "아, 내가 안 가르쳐줘도 알아서 피해가는 거구나"
- **40–50분 │ 인스타 콘텐츠 보너스**
  - 프롬프트 예: `"오늘 만든 운동 프로그램 중 하나를 골라서, 인스타그램 캡션을 내 PT샵 톤으로 써줘. 운동 효과 + 주의점 + 해시태그 포함"`
  - **이중 wow.** "운동도 짜고 마케팅도 해주네"

**막히면 강사 힌트:**
- "프로그램이 너무 빡세요" → "CLAUDE.md에 '회원 운동 강도는 항상 RPE 6 이하'라고 적어봅시다" (어려우면 그냥 "강도는 부드럽게")
- "회원 정보를 매번 설명하기 귀찮아요" → "그래서 members 폴더에 파일로 저장한 거예요. 다음엔 회원 이름만 말해도 알아요"

**백업 프로젝트:** 1주일 식단 추천 마크다운 생성기 (회원 알레르기·선호 입력 → 식단표 출력)

## 4. 트러블슈팅 인덱스

| 증상 | 원인 | 즉답 |
|------|------|------|
| `command not found: claude` | PATH 미반영 | 터미널 재시작. 안 되면 `source ~/.zshrc` |
| `npm: command not found` | Node.js 미설치 | nodejs.org에서 LTS 설치, 노트북 재시작 |
| 권한 오류 (EACCES) | 글로벌 설치 권한 | `sudo npm install -g …` 한 번만 |
| 윈도우에서 명령 안 먹힘 | CMD를 쓰고 있음 | PowerShell로 통일 |
| Claude가 엉뚱한 폴더에 파일 만듦 | 현재 디렉토리 모름 | `pwd`로 확인 후 "지금 이 폴더 안에" 명시 |
| 인증/로그인 오류 | API 키 또는 계정 문제 | 사전 안내 메일대로 재로그인 |
| 결과가 영어로 나옴 | 명시 안 함 | "한국어로" 한마디 추가 |
| 학습자가 좌절 | 자기 비교 | 강사가 본인 6개월 전 얘기 꺼낸다. **절대 "쉬워요"라고 말하지 말 것.** |
| 와이파이 끊김 | — | 모바일 핫스팟 미리 켜둠 |
| 한 명만 너무 빨리 끝남 | — | "두 번째 버전 다른 톤으로 만들어보세요" |

## 5. 리스크 & 백업 플랜

| 리스크 | 발생 시 |
|--------|---------|
| 설치에서 한 명이 30분 넘게 막힘 | 강사 노트북을 빌려준다. 그날은 강사 노트북으로 실습. 설치는 워크숍 후 1:1로 따로 진행 약속. |
| 두 학습자 디지털 친숙도 격차 큼 | 빠른 사람을 "보조 강사"로 만든다. ("작가님, 트레이너님 좀 도와드릴 수 있어요?") |
| 학습자가 "이거 너무 어려워요" 모드 | 즉시 멈추고 가장 단순한 성공 1개로 후퇴. ("그럼 그냥 ‘안녕'이라고만 써달라고 해봅시다") |
| 에너지 처짐 (특히 14시 이후 워크숍) | 휴식 5분 추가, 간식 투입 |
| Claude API가 오늘 느림/장애 | 강사가 미리 만든 스크린샷 자료로 시연 → 학습자 실습은 단순화 |

## 7. 강사 자기 점검

워크숍 끝나고 강사 본인이 답해보기:
1. 두 학습자가 각자 손에 든 산출물이 있는가?
2. 두 사람 모두 적어도 한 번은 "와" 했는가?
3. 두 사람 모두 다음 한 걸음을 알고 돌아갔는가?
4. 내가 키보드를 잡은 적이 있는가? (잡았다면 다음번엔 줄이기)
5. 내가 "쉬워요"라고 말한 적 있는가? (있었다면 다음번엔 안 말하기)
