오늘 우리가 함께 할 일

학습자

  • 작가님 — 베스트셀러 작가, 자기 브랜드 팟캐스트 운영
  • 트레이너님 — 1인 PT샵 운영, 회원 관리·프로그램 작성·콘텐츠 제작

오늘 끝나면 할 수 있는 것

  1. Claude Code가 무엇인지 한 문장으로 설명할 수 있다
  2. 내 노트북에서 Claude Code로 내 일과 관련된 작업 한 가지를 시켜본 경험이 있다
  3. "하네스"를 내 언어로 설명하고, 내 일에 적용할 첫 아이디어를 가지고 돌아간다

오늘의 약속 3가지

1. 정답 없습니다. 막히면 손드세요. 막힌 게 가장 좋은 학습 순간입니다.
2. 오늘 코드 한 줄도 직접 안 쓸 수도 있어요. 그게 이 도구의 핵심입니다.
3. 오늘 끝나고 자기 일에 쓸 수 있는 작은 거 하나 만들어 가는 게 목표입니다.

0–15분 · 오프닝

안녕하세요, 저도 비전공자입니다

아이스브레이커 질문

"요즘 일하면서 '아 이거 누가 대신 해줬으면 좋겠다' 싶은 일 한 가지만 말씀해주세요."

이 답이 90분 후 실습 프로젝트의 씨앗이 됩니다.

진행 흐름
  1. (3분) 환영 인사. "저도 비전공자입니다" 반드시 선언. 6개월 전 본인의 변화 1가지 짧게 공유.
  2. (6분) 두 학습자 자기소개 + 위 질문. 답변을 화이트보드/노션에 받아 적는다.
  3. (4분) 오늘의 약속 3가지 공유.
  4. (2분) 오늘 타임라인 한 장으로 보여주기.

강사 멘트 예시:

"여러분, 이거 코딩 배우는 워크숍이 아니에요. 코딩을 안 하고도 컴퓨터한테 일을 시키는 법을 배우는 워크숍입니다. 저도 여전히 코드 못 짭니다. 근데 이거 배우고 나서 제 팟캐스트 대본 정리하는 데 쓰던 2시간이 10분이 됐어요."

주의: "프로그래밍" "터미널" 같은 단어는 첫 15분 안에는 쓰지 말 것.
15–30분 · 개발 기초

개발, 무서운 게 아닙니다

"코딩도 모르는데 이걸 어떻게 해요?" — 저도 처음에 똑같은 생각이었어요. 그런데 알고 보니, 알아야 할 건 생각보다 훨씬 적었습니다.

1. 터미널 = 컴퓨터에게 글자로 말하는 창

여러분이 평소에 마우스로 폴더를 열고, 파일을 더블클릭하죠? 터미널은 똑같은 일을 글자로 하는 것뿐입니다.

마우스로 하면터미널에 쓰면
바탕화면에 폴더 만들기mkdir workshop
그 폴더 안으로 들어가기cd workshop
파일 목록 보기ls (맥) / dir (윈도우)

오늘 우리가 쓸 PowerShell이 바로 이 "터미널"입니다. 까만 창(또는 파란 창)이 무섭게 보여도, 그냥 컴퓨터랑 문자 주고받는 카톡방이라고 생각하세요.

2. 코드 = 컴퓨터가 읽는 레시피

요리 레시피처럼 "이 재료를 이 순서로 이렇게 해라"를 컴퓨터 언어로 적어둔 것이 코드예요.

근데 이제 바이브코딩 시대에는? 레시피를 우리가 직접 안 씁니다.
"이런 요리 만들어줘"라고 말하면 AI가 레시피를 써줍니다.

3. 웹사이트는 이렇게 돌아갑니다

내 컴퓨터 (브라우저)  ←→  인터넷  ←→  서버 (다른 컴퓨터)
    "보여주는 쪽"                      "일하는 쪽"
용어쉬운 말비유
프론트엔드 눈에 보이는 화면 식당의 홀 — 손님이 보는 메뉴판, 인테리어
백엔드 눈에 안 보이는 뒷단 식당의 주방 — 주문 받고 요리하고 재고 관리
서버 24시간 켜져있는 컴퓨터 식당 건물 자체 — 홀과 주방이 들어있는 곳
배포 만든 걸 인터넷에 올리기 식당 오픈 — 손님이 찾아올 수 있게 문 여는 것

4. 파일과 폴더 — 이미 알고 계신 거예요

코딩이라고 하면 뭔가 특별한 것 같지만, 결국 파일을 만들고 고치는 일입니다.

  • 웹사이트 = HTML 파일 + CSS 파일 + JavaScript 파일을 폴더에 넣은 것
  • 앱 = 비슷한 파일들을 폰에서 돌아가게 묶은 것
  • 우리가 워드 파일 만들 줄 알면, 원리는 같습니다

5. Git과 GitHub — 작업의 타임머신 + 공유 드라이브

코딩을 하다 보면 "아까 그게 더 나았는데…" 하는 순간이 옵니다. 이런 문제를 해결해주는 프로그램이 있어요.

Git, GitHub, 커밋 — 뭐가 뭔가요?

이름정체하는 일비유
Git 내 컴퓨터에 설치하는 프로그램 파일 변경 이력을 추적·관리 워드의 "변경 내용 추적" — 근데 훨씬 강력한 버전
GitHub 인터넷 웹사이트/서비스 (github.com) Git으로 관리한 파일을 클라우드에 올리고 공유 코드 전용 구글 드라이브 — 누가 뭘 고쳤는지 다 보임
커밋 (commit) Git 프로그램의 명령어 중 하나 저장 + "여기까지 했음" 메모 남기기 게임의 세이브 포인트

흐름으로 보면 이렇습니다:

Git (내 컴퓨터 프로그램)   →  세이브(커밋) 찍고 이력 관리
        ↓ 올리기 (push)
GitHub (인터넷 서비스)    →  백업 + 공유 + 공개

GitHub로 할 수 있는 일들:

  • 내가 만든 코드를 안전하게 백업 (노트북 고장나도 OK)
  • 작업 이력을 보고 아무 시점으로 되돌리기
  • 만든 웹사이트를 GitHub에서 바로 무료로 공개 (GitHub Pages)
  • 다른 사람과 같은 프로젝트를 동시에 작업
  • 전 세계 오픈소스 코드를 가져다 쓰기

오늘 당장 쓰진 않지만, 바이브코딩을 계속하면 자연스럽게 만나게 돼요. 재밌는 건, Claude Code가 Git을 대신 써준다는 거예요. "이거 세이브해줘"라고 하면 알아서 커밋을 해줍니다. Git 명령어를 외울 필요는 전혀 없고, Claude가 "커밋할까요?" 하고 물어볼 때 "아 게임 세이브하겠다는 거구나" 하면 됩니다.

6. 그래서 바이브코딩이란?

"내가 코드를 쓰는 게 아니라, AI에게 내 의도를 말하면 AI가 코드를 쓴다."

위의 모든 개념 — 터미널, 코드, 서버, Git — 을 여러분이 다 알 필요는 없어요. AI가 알고 있거든요. 여러분은 "뭘 만들고 싶은지"만 알면 됩니다. 나머지는 Claude가 합니다.

오늘 이 6가지만 머릿속에 넣고 가면 충분합니다. 지금부터 실제로 해볼 거예요.

이 섹션의 핵심은 "두려움 제거"입니다
  • 기술 용어를 가르치는 게 목적이 아닙니다. "아, 저것도 별거 아닌 거구나" 느끼게 하는 게 목적.
  • 각 개념을 30초 이내로 설명. 깊이 들어가지 말 것.
  • 학습자가 "그래서 코딩 배워야 하는 거 아니에요?"라고 물으면 → "아뇨, 오늘 코드 한 줄도 안 쓸 거예요. 이건 그냥 지도 보는 거예요. 지도를 보고 직접 운전하는 건 Claude가 합니다."
  • 강사 본인이 처음에 겁났던 경험을 솔직하게 공유하면 효과적: "저도 터미널이라는 단어만 들어도 도망가고 싶었어요"

진행 흐름 (15분)

  1. (2분) "사실 저도 이런 거 하나도 몰랐어요" — 강사 경험담으로 시작
  2. (10분) 6가지 개념을 화면 같이 보면서 빠르게 훑기. 각 항목 1-2분. Git/GitHub는 "오늘 안 쓰지만 곧 만날 거예요" 톤으로 가볍게.
  3. (1분) "바이브코딩이란?" — 핵심 메시지 전달. "여러분은 뭘 만들고 싶은지만 알면 됩니다."
  4. (2분) 질문. 보통 "진짜 코드 안 써도 돼요?" → "네, 진짜요. 지금부터 보여드릴게요."
주의: 여기서 너무 오래 머물면 학습자가 "역시 어려운 거구나" 모드에 빠진다. 빠르고 가볍게, 핵심은 "이 정도면 충분하다"는 자신감 심어주기.
30–50분 · Claude Code란?

ChatGPT랑 뭐가 달라요?

ChatGPT가 채팅창에 답해주는 친구라면, Claude Code는 내 컴퓨터에 들어와서 직접 일을 해주는 친구예요. 파일을 만들고, 폴더를 정리하고, 글을 고치고, 자료를 모아옵니다.

ChatGPT       →  말로만 답함     →  복사 붙여넣기는 내가
Claude Code   →  내 컴퓨터로 들어옴 →  파일·폴더를 직접 만짐

지금부터 보실 3가지

  1. Wow #1 — 폴더와 파일을 자연어로 만들어보기
  2. Wow #2 — 다운로드 폴더의 PDF들을 한 번에 정리하기
  3. Wow #3 — 일부러 모호하게 시켜보기 ("이거 좀 정리해줘")
라이브 시연 가이드
Wow명령포인트
#1 (5분) "내 데스크탑에 'workshop' 폴더 만들고 그 안에 hello.txt 파일 만들어서 '안녕하세요'라고 써줘" 폴더가 진짜 생기는 걸 두 사람이 직접 보게 한다
#2 (5분) "내 다운로드 폴더에 있는 PDF 파일들의 제목만 모아서 마크다운 표로 정리해줘" 여기서 "엇" 반응이 나옴
#3 (5분) "이거 좀 정리해줘" 일부러 실패. Claude가 되묻는 모습 보여줌. 교훈: "Claude는 마법이 아니라 똑똑한 동료. 동료한테 지시하듯 구체적으로."

(5분) Q&A. 보통 첫 질문은 "이거 돈 들어요?" — 솔직하게 답한다.

주의: 너무 화려한 거 보여주지 말 것. "나는 절대 못 해" 모드 들어가면 망친다. 일상적이고 작은 것 위주.
50–80분 · 설치 + Hello World

이제 직접 해봅시다 (윈도우)

Step 1. Node.js 설치 (이미 되어있으면 건너뛰기)

Claude Code를 쓰려면 먼저 Node.js라는 프로그램이 필요합니다. 한 번만 설치하면 돼요.

  1. nodejs.org 접속 → LTS 버전 다운로드
  2. 설치 마법사에서 모두 "Next" → "Install" → "Finish"
  3. 설치 확인: PowerShell을 열고 아래를 입력
node --version

v20.x.x 같은 숫자가 나오면 성공!

중요: 윈도우에서는 반드시 PowerShell을 사용합니다. CMD(명령 프롬프트)가 아닙니다! 작업표시줄 검색에 "PowerShell" 을 치면 나옵니다.

Step 2. Claude Code 설치

PowerShell에 한 줄만 입력하면 됩니다.

npm install -g @anthropic-ai/claude-code

Step 3. 첫 실행

설치가 끝나면 그대로 이렇게 쳐봅니다.

claude

환영 화면이 나오면 성공입니다. 여기까지 오면 박수!

Hello World — 자기 일과 살짝 연결

  • 작가님: "오늘 워크숍 참가 후기를 3문단으로 써줘. 톤은 친근하게."
  • 트레이너님: "오늘 시작한 새로운 도구 워크숍 후기를 인스타 캡션 스타일로 써줘. 해시태그도 포함해서."

각자 결과를 옆 사람한테 보여주세요. 첫 공유 모먼트입니다.

이 35분이 워크숍의 운명을 결정합니다.
  • (15분) 학습자 한 명씩 옆에 붙어서 설치. 다른 학습자는 사용 사례 메모를 적어둔다.
  • (5분) 두 사람이 동시에 claude 입력 → 환영 화면 → 박수! → "여기가 제일 큰 산이었어요. 지금부터는 쉽습니다" 선언.
  • (10분) Hello World. 두 사람이 거의 동시에 끝나도록 페이스 조절.

윈도우 설치 시 흔한 막힘 (자세한 대응은 트러블슈팅 참조)

  • PowerShell 여는 법 모름 → 작업표시줄 검색에 "PowerShell" 입력
  • node --version이 안 됨 → Node.js 설치 안 됐거나 PowerShell을 설치 전에 열어둔 것. PowerShell 닫고 재시작.
  • npm에서 권한 오류 → 관리자 권한 PowerShell로 재시도 (아래 트러블슈팅 참조)
  • 설치 중 빨간 글씨(WARN) → 대부분 무시해도 됨. ERROR가 아니면 OK.

강사 멘트 (막힐 때):

"지금 막히신 거 진짜 좋은 거예요. 이 막힘을 한 번 겪어두면 혼자 할 때도 안 무서워져요. 자, 같이 해결해봅시다."

절대 "쉬워요"라고 말하지 말 것. 한 명이 빨리 끝나면 "두 번째 버전 다른 톤으로 만들어보세요"로 시간 벌기.

놀라운 사실: 같은 AI 모델이라도 하네스 설계만 개선하면 성능이 +26% 향상된다는 연구가 있어요. 더 좋은 AI를 기다리는 것보다, 더 좋은 환경을 만들어주는 게 먼저입니다.

재료비유하는 일

오늘의 한 장 요약

Claude Code가 뭐였더라?

ChatGPT가 채팅창에서 답하는 친구라면, Claude Code는 내 컴퓨터 안에 들어와서 직접 일을 해주는 친구입니다. 폴더를 만들고, 파일을 고치고, 글을 써주고, 자료를 정리합니다.

오늘 손에 익힌 명령 5개

  1. claude — 시작하기
  2. "~~ 폴더 만들고 ~~ 파일 만들어줘" — 파일 생성
  3. "이 글을 ~~ 톤으로 다시 써줘" — 내 스타일로 변경
  4. "CLAUDE.md에 ~~ 한 줄 추가해줘" — 하네스 업데이트
  5. Ctrl+C 두 번 — 빠져나오기

핵심 개념: 하네스

  • CLAUDE.md = Claude한테 주는 "나는 누구고 어떻게 일하는 사람인지" 안내서
  • 처음엔 5–10줄로 시작. 막힐 때마다 한 줄씩 추가.
  • 이게 쌓일수록 Claude가 점점 "내 동료"가 됩니다.

막혔을 때 시도할 3가지 (순서대로)

  1. 더 구체적으로 다시 말해보기 — 사람한테 시킬 때처럼
  2. 새 대화로 다시 시작 (Ctrl+C 두 번 → 다시 claude)
  3. 강사한테 카톡 — 막힘은 학습입니다
